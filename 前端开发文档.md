## 3. 核心基础规范

### 3.1 类型定义 (TypeScript)

在 `src/types/api.d.ts` 中定义与后端约定的**统一响应体**：

```
// 统一 API 响应结构
export interface ApiResponse<T = any> {
  code: number;
  message: string;
  data: T;
  ts: number;
}

// 分页元数据结构
export interface PaginationMeta {
  total: number;
  page: number;
  page_size: number;
  pages: number;
}

// 分页响应结构
export interface PaginatedData<T> {
  list: T[];
  pagination: PaginationMeta;
}
```

### 3.2 API 请求封装 (Axios)

在 `src/api/client.ts` 中处理统一业务逻辑：自动剥离 `data` 层，全局拦截 `code !== 0` 的错误。

```
import axios from 'axios';
import { toast } from 'react-hot-toast'; // 或使用 shadcn 的 toast

export class ApiError extends Error {
  constructor(public code: number, message: string) {
    super(message);
    this.name = 'ApiError';
  }
}

export const client = axios.create({
  baseURL: import.meta.env.VITE_API_BASE_URL || '/api/v1',
  timeout: 10000,
});

client.interceptors.response.use(
  (response) => {
    const res = response.data;
    // 业务逻辑错误
    if (res.code !== 0) {
      toast.error(res.message || '请求失败');
      return Promise.reject(new ApiError(res.code, res.message));
    }
    // 直接返回业务数据层
    return res.data;
  },
  (error) => {
    // HTTP 层级错误 (如 500, 404)
    const msg = error.response?.data?.message || error.message || '网络错误';
    toast.error(msg);
    return Promise.reject(error);
  }
);
```

### 3.3 数据缓存与同步 (TanStack Query)

前端**禁用原生的 `useEffect` 去拉取数据**，所有数据请求必须通过 React Query 封装为 Custom Hook，位于 `src/hooks/`。

优势：自带 Loading/Error 状态，自带请求去重，自带定时轮询。

```
// 示例: src/hooks/useStockQuote.ts
import { useQuery } from '@tanstack/react-query';
import { stocksApi } from '@/api/stocks';

export function useStockQuote(tsCode: string) {
  return useQuery({
    queryKey: ['quote', tsCode],
    queryFn: () => stocksApi.getQuote(tsCode),
    // 交易时段 (简单判定)，这里可结合后端 is_trading 字段优化
    refetchInterval: (data) => (data?.is_trading ? 3000 : false), 
    staleTime: 3000, // 3秒内不重复发请求
    refetchOnWindowFocus: true, // 用户切回页面时立即刷新行情
  });
}
```

### 3.4 全局状态管理 (Zustand)

由于 React Query 接管了所有“服务端数据”，Zustand 只需管理少量的“客户端偏好”，如侧边栏收起、暗黑模式、红绿配色习惯等。

```
// src/store/useAppStore.ts
import { create } from 'zustand';
import { persist } from 'zustand/middleware';

interface AppState {
  theme: 'light' | 'dark';
  colorMode: 'red-up-green-down' | 'green-up-red-down'; // A股习惯 vs 国际习惯
  toggleTheme: () => void;
  setColorMode: (mode: 'red-up-green-down' | 'green-up-red-down') => void;
}

export const useAppStore = create<AppState>()(
  persist(
    (set) => ({
      theme: 'light',
      colorMode: 'red-up-green-down', // 默认 A股习惯：红涨绿跌
      toggleTheme: () => set((state) => ({ theme: state.theme === 'light' ? 'dark' : 'light' })),
      setColorMode: (mode) => set({ colorMode: mode }),
    }),
    { name: 'astock-preferences' }
  )
);
```

## 5. 核心业务组件设计

### 5.1 K线图表 (ECharts 封装)

K 线图是整个项目最复杂的前端组件。采用 `echarts-for-react` 进行轻量级封装。

**规范要点**：

1. **数据切片**：`dataset` 模式，前端不要手动拼装 series，将 API 返回的数组转为二维数组交由 echarts 处理。
2. **主题跟随**：图表背景、网格线需根据 `useAppStore` 的暗黑模式自动切换。
3. **红绿配置**：阳线/阴线颜色需读取用户的 `colorMode` 配置。

```
// src/components/charts/KLineChart.tsx
import ReactECharts from 'echarts-for-react';
import { useAppStore } from '@/store/useAppStore';

export const KLineChart = ({ data }) => {
  const { theme, colorMode } = useAppStore();
  
  // A股习惯：红涨绿跌
  const upColor = colorMode === 'red-up-green-down' ? '#ef4444' : '#10b981';
  const downColor = colorMode === 'red-up-green-down' ? '#10b981' : '#ef4444';

  // 转换数据为 ECharts dataset 格式
  const sourceData = data.map(item => [
    item.date, item.open, item.close, item.low, item.high, item.vol, item.pct_chg
  ]);

  const option = {
    backgroundColor: 'transparent',
    tooltip: { trigger: 'axis', axisPointer: { type: 'cross' } },
    dataset: { source: sourceData },
    grid: [
      { left: '5%', right: '5%', top: '5%', height: '60%' }, // K线主图
      { left: '5%', right: '5%', top: '70%', height: '20%' } // 成交量副图
    ],
    xAxis: [
      { type: 'category', gridIndex: 0 },
      { type: 'category', gridIndex: 1, show: false } // 副图 X 轴隐藏
    ],
    yAxis: [
      { scale: true, gridIndex: 0 }, // 价格
      { scale: true, gridIndex: 1, splitNumber: 2 } // 成交量
    ],
    dataZoom: [
      { type: 'inside', xAxisIndex: [0, 1], start: 50, end: 100 },
      { type: 'slider', xAxisIndex: [0, 1], bottom: '0%' }
    ],
    series: [
      {
        name: 'K线',
        type: 'candlestick',
        encode: { x: 0, open: 1, close: 2, low: 3, high: 4 },
        itemStyle: {
          color: upColor, color0: downColor,
          borderColor: upColor, borderColor0: downColor
        }
      },
      {
        name: '成交量',
        type: 'bar',
        xAxisIndex: 1,
        yAxisIndex: 1,
        encode: { x: 0, y: 5 },
        // 根据开盘价和收盘价决定成交量柱子颜色
        itemStyle: {
          color: (params) => {
            const row = params.data;
            return row[2] > row[1] ? upColor : downColor;
          }
        }
      }
    ]
  };

  return <ReactECharts option={option} theme={theme} style={{ height: '500px' }} />;
};
```

### 5.2 格式化与颜色工具 (`src/utils/format.ts`)

为了保证整个应用 UI 统一，所有价格、涨跌幅、市值的展示必须经过工具函数：

```
// 涨跌幅格式化，带正负号，如 "+1.23%", "-0.56%"
export const formatPct = (val: number | null | undefined): string => {
  if (val == null) return '--';
  return val > 0 ? `+${val.toFixed(2)}%` : `${val.toFixed(2)}%`;
};

// 提取涨跌颜色 CSS 类名 (适配 Tailwind)
export const getChangeColorClass = (val: number, mode: 'red-up-green-down' = 'red-up-green-down') => {
  if (!val || val === 0) return 'text-gray-500 dark:text-gray-400';
  if (mode === 'red-up-green-down') {
    return val > 0 ? 'text-red-500' : 'text-green-500';
  }
  return val > 0 ? 'text-green-500' : 'text-red-500';
};

// 大额数字格式化：万、亿
export const formatAmount = (val: number): string => {
  if (!val) return '--';
  if (val >= 100000000) return `${(val / 100000000).toFixed(2)}亿`;
  if (val >= 10000) return `${(val / 10000).toFixed(2)}万`;
  return val.toFixed(2);
};
```

## 6. 样式与 UI 规范

### 6.1 Tailwind 核心约定

- 禁止手写长篇 Custom CSS，绝大部分样式需使用 Tailwind 类名。
- 采用暗黑模式支持：凡是涉及到背景和文本颜色的，必须写 `dark:` 变体。例如 `bg-white dark:bg-slate-900`。
- 卡片风格统一：`rounded-xl border border-slate-200 dark:border-slate-800 bg-white dark:bg-slate-900 shadow-sm p-4`。

### 6.2 交互规范

- **加载状态**：页面的全量加载使用骨架屏 (`Skeleton`)；局部按钮的提交操作使用带 Spinner 的 `Button`。
- **空状态**：表格或列表无数据时，统一展示插画加提示文案的 `<EmptyState />` 组件。
- **交互反馈**：所有的添加、删除、修改操作成功后，必须调用 `toast.success`，并通过 `queryClient.invalidateQueries` 刷新本地数据。

## 7. 环境变量与部署

前端由 Vite 构建，使用环境变量来区分开发和生产环境。

### 7.1 本地开发 (.env.development)

```
# Vite 本地开发时，会被 vite.config.ts 中的 proxy 拦截转发给后端 8000 端口
VITE_API_BASE_URL=/api/v1
```

**`vite.config.ts` 代理配置**：

```
import { defineConfig } from 'vite';
import react from '@vitejs/plugin-react';
import path from 'path';

export default defineConfig({
  plugins: [react()],
  resolve: {
    alias: { '@': path.resolve(__dirname, './src') },
  },
  server: {
    port: 3000,
    proxy: {
      '/api': {
        target: 'http://localhost:8000', // 后端地址
        changeOrigin: true,
      }
    }
  }
});
```

### 7.2 生产部署 (.env.production)

如果前端打包为静态文件（通过 Nginx 部署）或者由后端直接代理：

```
VITE_API_BASE_URL=/api/v1
```

生产环境使用 Docker 部署时，前端镜像推荐使用 Nginx 托管构建产物：

```
# frontend/Dockerfile
FROM node:18-alpine AS builder
WORKDIR /app
COPY package*.json ./
RUN npm install
COPY . .
RUN npm run build

FROM nginx:alpine
COPY --from=builder /app/dist /usr/share/nginx/html
COPY nginx.conf /etc/nginx/conf.d/default.conf
EXPOSE 80
CMD ["nginx", "-g", "daemon off;"]
```

相应的 Nginx 配置中需将 `/api` 请求转发至 backend 服务（与 docker-compose 对应）。
