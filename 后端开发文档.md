## 1. 整体架构

### 1.1 架构图（文字版）

```
┌─────────────────────────────────────────────────────────┐
│                     多数据源（轮动）                      │
│  Sina财经 · BaoStock · AKShare · 同花顺THS               │

│  │  DataSourceManager (优先→降级自动切换+缓存)       │   │
│  └─────────────────────────────────────────────────┘   │

└──────────────────────────────────────────┘
```

### 1.2 设计原则

| 原则 | 说明 |
|------|------|
| **单一职责** | 采集、存储、服务、展示各自独立，互不耦合 |
| **渐进增强** | 先跑通核心链路（日行情），再逐步加数据源 |
| **本地优先** | 数据落本地库，不依赖任何付费云服务 |
| **接口幂等** | 所有数据写入操作可重复执行不产生脏数据（UPSERT） |

## 2. 技术栈选型

| 层次 | 选型 | 理由 |
|------|------|------|
| 数据采集 | `AKShare` + `httpx` + `BeautifulSoup4` + `baostock` | 多源轮动，免费、无需Token |
| 数据源轮动 | `DataSourceManager` (自定义) | 优先级切换+故障自动fallback |
| 任务调度 | `APScheduler`（内嵌FastAPI） | 无需额外队列服务，够轻 |
| 后端框架 | `FastAPI` + `SQLAlchemy 2.0` | 自动生成文档、async支持、类型安全 |
| 数据库 | `SQLite`（开发）→ `PostgreSQL`（生产） | 无缝切换，开发零配置 |
| 缓存 | `Redis`（可选，降级为内存字典） | 行情缓存、接口限速 |
| 前端框架 | `React 18` + `Vite` + `TypeScript` | 生态成熟、构建快 |
| CSS | `TailwindCSS` | 无需写样式文件 |
| 图表 | `ECharts`（via `echarts-for-react`） | K线图首选 |
| 部署 | `Docker Compose` | 一键启动全部服务 |

## 4. 数据库设计

> 所有表使用 `UPSERT`（INSERT OR REPLACE / ON CONFLICT DO UPDATE）写入，保证采集任务可重跑。

### 4.1 stocks — 股票基础信息

```sql
CREATE TABLE stocks (
    ts_code      VARCHAR(12) PRIMARY KEY,  -- "600000.SH"
    name         VARCHAR(50)  NOT NULL,     -- "浦发银行"
    market       VARCHAR(10)  NOT NULL,     -- "SH" / "SZ" / "BJ"
    industry     VARCHAR(50),               -- 申万一级行业
    list_date    DATE,
    delist_date  DATE,                      -- NULL = 在市
    total_share  NUMERIC(20,2),             -- 总股本（万股）
    float_share  NUMERIC(20,2),             -- 流通股本（万股）
    updated_at   TIMESTAMP DEFAULT NOW()
);
```

## 5. 数据采集层设计

### 5.1 采集任务总览

> **v2.1 更新**：数据采集已改用轮动数据源（`DataSourceManager`），单个 API 不可用时自动 fallback。
> 数据源优先级：Sina财经 → BaoStock → AKShare → 同花顺 THS

| 任务名 | 触发时间 | 数据源 | 写入表 | 说明 |
|--------|----------|--------|--------|------|
| `sync_stock_list` | 每周一 09:00 | AKShare `stock_info_a_code_name()` | stocks | 全量股票元数据 (~5200只) |
| `sync_daily_price` | 工作日 17:00 | Sina `stock_zh_a_daily()` / BaoStock | daily_prices | 当日全量行情 |
| `sync_realtime_quote` | 工作日 9:30-15:00 每3分钟 | Sina `hq.sinajs.cn` | Redis（不落库） | 实时行情快照 |
| `sync_financials` | 每周一 08:00 | AKShare `stock_financial_abstract_ths()` | financials | 最新季报/年报 |
| `sync_announcements` | 工作日 08:30 / 18:00 | AKShare `stock_notice_report()` | announcements | 增量公告 |
| `sync_news` | 每30分钟 | Sina财经 → 同花顺THS (轮动) | news | 增量新闻 |
| `sync_indices` | 实时（API调用时） | Sina指数API / BaoStock | 缓存 | 四大指数行情 |
| `sync_sectors` | 实时（API调用时） | AKShare `stock_board_industry_summary_ths()` | 缓存 | 90个行业板块数据 |

### 5.2 采集器基类

```python
# crawlers/base.py
class BaseCrawler:
    """
    所有采集器继承此类，提供：
    - 统一日志
    - 异常捕获 + 重试（最多3次，指数退避）
    - UPSERT 写入封装
    - 采集结果统计上报
    """

    def fetch(self) -> list[dict]:
        """子类实现：拉取原始数据，返回标准化字典列表"""
        raise NotImplementedError

    def run(self):
        for attempt in range(3):
            try:
                records = self.fetch()
                self.upsert(records)
                logger.info(f"{self.__class__.__name__} 完成，写入 {len(records)} 条")
                return
            except Exception as e:
                wait = 2 ** attempt
                logger.warning(f"第{attempt+1}次失败：{e}，{wait}s后重试")
                time.sleep(wait)
        logger.error(f"{self.__class__.__name__} 最终失败")
```

### 5.3 日行情采集器示例

```python
# crawlers/daily_price.py
class DailyPriceCrawler(BaseCrawler):

    def fetch(self) -> list[dict]:
        # 获取今日交易的全量股票代码
        codes = db.query(Stock.ts_code).filter(Stock.delist_date == None).all()
        result = []
        for ts_code, in codes:
            symbol = ts_code.split(".")[0]  # "600000.SH" -> "600000"
            df = ak.stock_zh_a_hist(
                symbol=symbol,
                period="daily",
                start_date=today_str,
                end_date=today_str,
                adjust="qfq"  # 前复权
            )
            if df.empty:
                continue
            row = df.iloc[0]
            result.append({
                "ts_code": ts_code,
                "trade_date": row["日期"],
                "open": row["开盘"],
                "high": row["最高"],
                "low": row["最低"],
                "close": row["收盘"],
                "pct_chg": row["涨跌幅"],
                "vol": row["成交量"],
                "amount": row["成交额"],
                "turnover_rate": row["换手率"],
            })
            time.sleep(0.1)  # 礼貌延迟
        return result

    def upsert(self, records):
        # PostgreSQL: INSERT ... ON CONFLICT DO UPDATE
        # SQLite:     INSERT OR REPLACE INTO ...
        stmt = pg_insert(DailyPrice).values(records)
        stmt = stmt.on_conflict_do_update(
            index_elements=["ts_code", "trade_date"],
            set_={k: stmt.excluded[k] for k in records[0] if k not in ("ts_code", "trade_date")}
        )
        db.execute(stmt)
        db.commit()
```

## 6. 后端 API 接口规范

### 6.0 统一规范

**Base URL**：`/api/v1`

**统一响应体**（所有接口均返回此结构）：

```json
{
  "code": 0,          // 0=成功, 非0=业务错误
  "message": "ok",   // 错误时含原因
  "data": {},        // 响应数据（对象或数组）
  "ts": 1717200000000 // 服务器时间戳（毫秒）
}
```

**错误码表**：

| code | 含义 |
|------|------|
| 0 | 成功 |
| 1001 | 参数校验失败 |
| 1002 | 资源不存在 |
| 1003 | 数据暂未就绪 |
| 5001 | 服务内部错误 |

**分页参数**（列表接口通用）：

```
?page=1&page_size=20
```

响应中带 `pagination` 字段：

```json
"pagination": {
  "total": 150,
  "page": 1,
  "page_size": 20,
  "pages": 8
}
```

### 6.1 股票基础信息接口

#### `GET /stocks` — 搜索/列表

**用途**：搜索栏模糊查询、行业筛选、市场筛选

**Query 参数**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `q` | string | 否 | 模糊搜索，匹配代码/名称/拼音首字母 |
| `market` | string | 否 | `SH` / `SZ` / `BJ` |
| `industry` | string | 否 | 申万一级行业，如 `银行` |
| `page` | int | 否 | 默认 1 |
| `page_size` | int | 否 | 默认 20，最大 100 |

**响应 `data`**：

```json
{
  "list": [
    {
      "ts_code": "600000.SH",
      "name": "浦发银行",
      "market": "SH",
      "industry": "银行",
      "list_date": "1999-11-10",
      "total_mv": 18500000.00,   // 总市值（万元），来自最新日行情
      "circ_mv":  15200000.00,   // 流通市值（万元）
      "pct_chg": -0.52           // 今日涨跌幅（%）
    }
  ],
  "pagination": { "total": 5200, "page": 1, "page_size": 20, "pages": 260 }
}
```

**缓存**：`stock:list:{q}:{market}:{industry}:{page}:{page_size}` → TTL **300s**（搜索结果）

### 6.2 行情接口

#### `GET /stocks/{ts_code}/quote` — 实时行情快照

**用途**：个股详情页顶部价格区域

**响应 `data`**：

```json
{
  "ts_code": "600000.SH",
  "name": "浦发银行",
  "price": 7.23,
  "pre_close": 7.27,
  "pct_chg": -0.55,
  "change": -0.04,
  "open": 7.28,
  "high": 7.30,
  "low": 7.20,
  "vol": 1250600,       // 成交量（手）
  "amount": 90523100.0, // 成交额（元）
  "turnover_rate": 0.52,
  "total_mv": 21230000.0,
  "circ_mv": 17420000.0,
  "quote_time": "2024-06-01T14:30:00+08:00", // 行情时间
  "is_trading": true    // 当前是否交易时段
}
```

**数据来源**：
- 交易时段（9:25-15:00）：读 Redis Key `rt:quote:{ts_code}`，由 `sync_realtime_quote` 每3分钟写入
- 非交易时段：读数据库 `daily_prices` 最新一条
- 若 Redis 不可用：降级读数据库

**缓存**：Redis `rt:quote:{ts_code}` → TTL **180s**（由采集任务主动刷新，接口直读不设额外缓存）

## 7. 缓存设计

### 7.1 缓存总表

| 缓存 Key 模式 | 内容 | TTL | 更新策略 | 降级行为 |
|--------------|------|-----|---------|---------|
| `stock:info:{ts_code}` | 个股基础信息 | 3600s | 定时任务写完后主动失效 | 直查DB |
| `stock:list:{hash}` | 搜索结果列表 | 300s | 被动过期 | 直查DB |
| `rt:quote:{ts_code}` | 实时行情快照 | 180s | 采集任务每3分钟主动写入 | 查DB最新日行情 |
| `rt:quote:all` | 全市场行情快照（Hash） | 180s | 全量覆盖写 | 逐个查DB |
| `market:overview` | 大盘概况 | 60s(交易时段)/3600s | 被动过期 + 定时刷新 | 聚合计算DB数据 |
| `market:sectors:{type}` | 板块涨跌幅 | 180s | 被动过期 | 直查DB聚合 |
| `kline:{ts_code}:{...}` | K线历史 | 1800s | 盘后采集完成后失效 | 直查DB |
| `financials:{ts_code}:{...}` | 财务数据 | 43200s | 财务采集完成后失效 | 直查DB |
| `ann:{ts_code}:{...}` | 公告列表 | 600s | 被动过期 | 直查DB |
| `news:list:{...}` | 新闻列表 | 120s | 被动过期 | 直查DB |

### 7.2 缓存层封装

```python
# cache.py
class Cache:
    """
    统一缓存抽象，支持 Redis（有则用）和内存字典（降级）。
    业务代码只调用 get/set/delete，不感知底层。
    """

    def get(self, key: str) -> Any | None: ...
    def set(self, key: str, value: Any, ttl: int): ...
    def delete(self, key: str): ...
    def delete_pattern(self, pattern: str): ...  # 批量失效，如 kline:600000.SH:*
```

### 7.3 缓存失效策略

**主动失效**（推荐，避免读到脏数据）：
- 日行情采集任务完成后，调用 `cache.delete_pattern("kline:{ts_code}:daily:*")`
- 财务数据写入后，调用 `cache.delete_pattern("financials:{ts_code}:*")`
- 股票信息更新后，调用 `cache.delete("stock:info:{ts_code}")`

**被动过期**（兜底）：所有 Key 均设置 TTL，防止缓存永不过期

**降级处理**：Redis 连接失败时，所有缓存操作静默失败，直接查数据库，系统继续运行

## 11. 多数据源轮动设计

> **v2.1 新增**。核心设计理念：一个数据源挂了，自动切下一个，保证服务不中断。

### 11.1 架构

```
DataSourceManager (单例)
  │
  ├─ [priority=1] SinaDataSource   ← 指数行情 / 个股+实时行情 / 财经新闻
  ├─ [priority=2] BaoStockSource   ← 历史K线（前复权）/ 指数日K
  └─ [priority=3] AKShareSource    ← 股票列表 / 行业板块 / 财务数据 / 新闻备选
```

### 11.2 故障切换机制

```python
def _try_sources(method, *args):
    for source in sorted_sources:       # 按 priority 排序
        if not source.is_available():   # 检查可用性（ping/登录）
            continue
        try:
            return source.method(*args) # 尝试调用
        except DataSourceError:
            continue                    # 失败 → 下一个
    raise DataSourceError("All failed") # 全部失败
```

### 11.3 各数据源能力矩阵

| 能力 | Sina | BaoStock | AKShare |
|------|:----:|:--------:|:-------:|

| 个股行情 | 实时快照 | 历史日K(前复权) | — |

### 11.4 缓存策略

数据源层自带缓存（`ds:*` 前缀），减少对外部 API 的重复请求：

| 缓存Key | TTL | 说明 |
|---------|-----|------|
| `ds:indices:*` | 60s | 指数行情 |
| `ds:quote:{ts_code}` | 30s | 个股实时行情 |
| `ds:sectors` | 180s | 行业板块 |
| `ds:news` | 120s | 财经新闻 |
| `ds:kline:{ts_code}:*` | 300s | K线历史数据 |

### 11.5 新增数据源

1. 在 `datasources/` 下新建文件，继承 `BaseDataSource`
2. 设置 `name` 和 `priority`
3. 实现需要的 `fetch_*` 方法
4. 在 `DataSourceManager.__init__` 中注册
5. 无需修改任何已有业务代码
